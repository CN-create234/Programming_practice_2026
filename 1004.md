# EOJ-1004 内存显示 - 逐行详细解释
```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
    string s;
    
    while (cin >> s) {
        // 判断是否包含小数点
        if (s.find('.') != string::npos) {
            // double 类型
            double d = atof(s.c_str());
            unsigned char* p = (unsigned char*)&d;
            
            for (int i = 0; i < sizeof(d); i++) {
                cout << hex << setw(2) << setfill('0') 
                     << (int)p[i] << " ";
            }
            cout << endl;
        } else {
            // int 类型
            int a = atoi(s.c_str());
            unsigned char* p = (unsigned char*)&a;
            
            for (int i = 0; i < sizeof(a); i++) {
                cout << hex << setw(2) << setfill('0') 
                     << (int)p[i] << " ";
            }
            cout << endl;
        }
    }

    return 0;
}
```
---

## 第一段：头文件和命名空间

```cpp
#include <bits/stdc++.h>
using namespace std;
```

- `#include <bits/stdc++.h>`：万能头文件，包含C++所有标准库（iostream, string, iomanip, cstdlib等）
- `using namespace std`：使用标准命名空间，可以直接写 `cout`、`cin`、`string`、`endl` 等，不用加 `std::` 前缀

---

## 第三段：循环读取输入

```cpp
    string s;
    
    while (cin >> s) {
```

- `string s`：声明一个字符串变量，用于存储每行输入的数值
- `while (cin >> s)`：循环读取输入，直到文件结束（EOF）
  - `cin >> s` 会跳过空白字符（空格、换行、制表符），读取下一个非空白字符串
  - 每行只有一个数值，所以每次读取一行正好

---

## 第四段：判断数据类型

```cpp
        // 判断是否包含小数点
        if (s.find('.') != string::npos) {
```

- `s.find('.')`：在字符串 `s` 中查找小数点 `.` 的位置
- `string::npos`：`find()` 函数在找不到时返回的特殊常量
- `!= string::npos`：如果找到了小数点（位置不是 npos），说明这是一个浮点数

**示例：**
```
"3.14" → find('.') = 1 → != npos → true → double
"3"    → find('.') = npos → false → int
"-1.0" → find('.') = 2 → != npos → true → double
```

---

## 第五段：处理 double 类型

```cpp
            // double 类型
            double d = atof(s.c_str());
            unsigned char* p = (unsigned char*)&d;//解释见后面
```

- `double d = atof(s.c_str())`：将字符串转换为 double 类型
  - `s.c_str()`：将 C++ string 转为 C 风格字符串（`const char*`）
  - `atof()`：ASCII to float/double，将字符串转为 double
  
- `unsigned char* p = (unsigned char*)&d`：
  - `&d`：获取变量 `d` 的起始内存地址，类型是 `double*`
  - `(unsigned char*)`：强制转换为 `unsigned char*` 类型
  - 这样 `p` 就指向 `d` 的第一个字节，可以通过 `p[0]`、`p[1]`... 访问每个字节

**为什么要强制转为 `unsigned char*`？**
- 不能直接用 `double*` 逐字节访问
- `unsigned char*` 可以按字节访问内存，且值范围是 0~255，能正确显示十六进制 00~FF

---

## 第六段：循环输出 double 的每个字节

```cpp
            for (int i = 0; i < sizeof(d); i++) {
                cout << hex << setw(2) << setfill('0') 
                     << (int)p[i] << " ";
            }
            cout << endl;
```

- `for (int i = 0; i < sizeof(d); i++)`：循环 `sizeof(d)` 次，即 double 占用的字节数（通常8次）

- `cout << hex`：设置输出格式为**十六进制**（后续所有整数输出都变成十六进制）

- `setw(2)`：设置输出最小宽度为2（来自 `<iomanip>`，已包含在万能头文件中）

- `setfill('0')`：宽度不足时用字符 `'0'` 填充

- `(int)p[i]`：
  - `p[i]` 是 `unsigned char` 类型，值范围 0~255
  - 直接输出 `unsigned char` 会显示为 ASCII 字符（如 65 显示 'A'）
  - 强制转为 `int` 后输出数字

- `" "`：每个字节后面输出一个空格

- `cout << endl`：该行结束后换行

**输出效果示例：**
```
1f 85 eb 51 b8 1e 09 40 
```

## 第七段：处理 int 类型

```cpp
        } else {
            // int 类型
            int a = atoi(s.c_str());
            unsigned char* p = (unsigned char*)&a;
            
            for (int i = 0; i < sizeof(a); i++) {
                cout << hex << setw(2) << setfill('0')

```

- `int a = atoi(s.c_str())`：将字符串转换为 int
  - `atoi()`：ASCII to integer，将字符串转为整数
  - `"123"` → 123，`"-456"` → -456

- 其余逻辑与 double 完全相同，只是 `sizeof(a)` 是4（通常 int 占4字节）

}

- `return 0`：程序正常结束，返回0给操作系统

## (unsigned char*)&d; 详细解释

---

## 一、拆解这个表达式

```cpp
(unsigned char*)&d;
```

从右到左看：

| 步骤 | 代码 | 含义 |
|------|------|------|
| 1 | `d` | 一个 double 变量 |
| 2 | `&d` | 取 d 的地址，类型是 `double*` |
| 3 | `(unsigned char*)` | 强制类型转换，变成 `unsigned char*` |

---

## 二、图解内存

假设 double d = 3.14，存储在地址 0x1000：

```
地址:  0x1000  0x1001  0x1002  0x1003  0x1004  0x1005  0x1006  0x1007
      ┌───────┬───────┬───────┬───────┬───────┬───────┬───────┬───────┐
数据: │  1f   │  85   │  eb   │  51   │  b8   │  1e   │  09   │  40   │
      └───────┴───────┴───────┴───────┴───────┴───────┴───────┴───────┘
```

### &d 的结果

```
&d = 0x1000
类型: double*
```

### (unsigned char*)&d 的结果

```
p = (unsigned char*)&d = 0x1000
类型: unsigned char*
```

**地址没变，但指针类型变了！**

## 三、指针类型不同，访问方式不同

| 指针类型 | 每次 +1 移动的字节数 | p[0] 访问的内容 | p[1] 访问的内容 |
|----------|---------------------|-----------------|-----------------|
| `double*` | 8 字节 | 整个 double（8字节） | 下一个 double（越界） |
| `unsigned char*` | 1 字节 | 第1个字节（0x1f） | 第2个字节（0x85） |

```cpp
double* dp = &d;
dp[0];   // 访问 8 字节，得到 3.14
dp[1];   // 访问下一个 8 字节（越界！）

unsigned char* p = (unsigned char*)&d;
p[0];    // 访问 1 字节，得到 0x1f
p[1];    // 访问 1 字节，得到 0x85
p[2];    // 访问 1 字节，得到 0xeb
p[3];    // 访问 1 字节，得到 0x51
p[4];    // 访问 1 字节，得到 0xb8
p[5];    // 访问 1 字节，得到 0x1e
p[6];    // 访问 1 字节，得到 0x09
p[7];    // 访问 1 字节，得到 0x40
```

## 五、为什么用 unsigned char？

### 原因1：char 可能是有符号的

```cpp
char* p = (char*)&d;
char c = p[0];    // 如果 p[0] = 0x85 (133)
// 在某些编译器上，char 是有符号的，c = -123（错误！）

unsigned char* p = (unsigned char*)&d;
unsigned char c = p[0];  // c = 133（正确！）
```

### 原因2：范围 0~255 正好对应十六进制 00~FF

```cpp
unsigned char 范围: 0 ~ 255
十六进制表示:     00 ~ FF
```

### 原因3：输出正确

```cpp
// 用 char 可能输出负数
cout << hex << (int)p[0];  // 可能输出 ffffff85

// 用 unsigned char 输出正确
cout << hex << (int)p[0];  // 输出 85
```

## 六、完整代码示例

```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
    double d = 3.14;
    
    // 获取字节指针
    unsigned char* p = (unsigned char*)&d;
    
    // 输出地址
    cout << "d 的起始地址: " << (void*)p << endl;
    
    // 输出每个字节
    cout << "逐字节内容: ";
    for (int i = 0; i < sizeof(d); i++) {
        cout << hex << setw(2) << setfill('0') 
             << (int)p[i] << " ";
    }
    cout << endl;
    
    // 证明 p[0] 到 p[7] 就是 d 的 8 个字节
    cout << "p[0] 的地址: " << (void*)&p[0] << endl;
    cout << "p[7] 的地址: " << (void*)&p[7] << endl;
    cout << "&d 的地址:   " << (void*)&d << endl;
    
    return 0;
}
```

---

## 七、一句话总结

`(unsigned char*)&d` 的作用是：

**获取变量 d 的起始地址，并将指针类型转为 `unsigned char*`，从而可以按字节访问 d 在内存中的每个字节内容。**
