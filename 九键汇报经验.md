### 2.2 重点关注：`match` 通配符命令 ⚠️

**这是最高频的测试点，也是最容易出错的点！**

#### 典型测试命令

```
add addddddnnnnnndddd 6
match a****d****n*d
```

#### ❌ 常见错误

**现象**：输出中出现大量重复的 `addddddnnnnnndddd`

```
> match a****d****n*d
addddddnnnnnndddd
addddddnnnnnndddd
addddddnnnnnndddd
addddddnnnnnndddd
addddddnnnnnndddd
...（重复多次）
```

**错误原因**：
- 多个 `*` 会产生多条不同的匹配路径
- 每条路径都可能到达同一个单词节点
- 所有路径的结果都存入 `result` 向量
- 导致同一个单词被重复收集多次

#### ✅ 正确做法

在 `matchPattern` 返回前进行**去重**：

```cpp
vector<string> Trie::matchPattern(const string& pattern) const {
    vector<string> result;
    collectByPattern(root_.get(), "", pattern, 0, result);

    // 去重 + 排序（字典序）
    sort(result.begin(), result.end());
    result.erase(unique(result.begin(), result.end()), result.end());
    
    return result;
}
```

### 2.3 通配符测试命令

| 测试类型 | 示例命令 | 检查点 |
|----------|----------|--------|
| 多个 `*` | `match a****d****n*d` | 是否重复输出 |
| `*` 开头/结尾 | `match *abc*` | 是否正确匹配 |
| `?` 匹配 | `match a?d` | 是否匹配单个字符 |
| 混合模式 | `match a?*d*` | 组合是否正确 |
| 全部匹配 | `match *` | 是否输出所有单词 |
| 无匹配 | `match zzz*` | 是否正确提示 |


## 三、汇报时 - 理解程度检验

### 3.1 数据结构问答

#### （1）`TrieNode` 结构体

```cpp
struct TrieNode {
    unordered_map<char, unique_ptr<TrieNode>> children;
    int freq = 0;       // 0=非单词，>0=单词+词频
    string word;        // 完整单词（冗余存储）
};
```

#### （2）`root` 是什么？

❌ **错误答案**：`root` 是字符串 `"abc"` 中的 `'a'`

✅ **正确答案**：`root` 是 **Trie 树的根节点**，是一个**空节点**（不存储任何字符），所有单词从 `root` 的子节点开始延伸。它就像一个"入口"，`'a'` 是 `root` 的子节点，而不是 `root` 本身。

```
正确的理解：
    [root] (空节点)  ← 这就是 root
     /  |  \
    a   b   c        ← root 的子节点
   / \
  n   ...
```

#### （3）`freq` 的含义和位置

| 问题 | 答案 |
|------|------|
| `freq` 含义 | 单词被选择的次数（词频） |
| `freq > 0` | 该节点是单词结尾 |
| `freq = 0` | 该节点不是单词（仅前缀） |
| `freq` 接在哪里 | 接在**单词最后一个字母对应的节点**上 |

```
示例：词库中有 "and"

[root] → [a] → [n] → [d] (freq=6, word="and")
                       ↑
                  freq 接在这里
```

### 3.2 基本操作问答

#### `delete` 删除操作的实现

```
1. 检查单词是否存在（wordSet_.count(word)）
2. 沿 Trie 路径查找单词末节点
3. 将末节点的 freq 设为 0（标记为非单词）
4. 清空 word 字段
5. 从 wordSet_ 和 wordFreq_ 中移除
6. 调用 invalidateCacheByString(word) 清除缓存
```

**关键理解**：节点本身**不删除**（物理保留），只标记 `freq = 0`。这样共享前缀的其他单词不受影响。

```
示例：删除 "and"
          [root] → [a] → [n] → [d] (freq=0, word="")
                               ↑
                          仅标记为0，节点保留
          
          "ando" 仍然可以正常访问：
          [root] → [a] → [n] → [d] → [o] (freq=4, word="ando")
                               ↑
                          共享 'd' 节点
```


## 四、最高频提问：`match` 通配符匹配原理 ⚠️

### 4.1 算法核心

```cpp
void collectByPattern(node, current, pattern, pos, result) {
    // 1. 终止条件：模式匹配完成
    if (pos == pattern.length()) {
        if (node->freq > 0) result.push_back(node->word);
        return;
    }
    
    char p = pattern[pos];
    
    if (p == '*') {
        // 2. * 匹配空串 → 跳过 *
        collectByPattern(node, current, pattern, pos + 1, result);
        // 3. * 匹配至少一个字符 → 吃掉字符，*保留
        for (auto& child : node->children)
            collectByPattern(child, current + ch, pattern, pos, result);
    } 
    else if (p == '?') {
        // 4. ? 匹配任意单个字符 → 遍历所有子节点，pos+1
        for (auto& child : node->children)
            collectByPattern(child, current + ch, pattern, pos + 1, result);
    } 
    else {
        // 5. 普通字符 → 精确匹配
        if (node->children.count(p))
            collectByPattern(node->children[p], current + p, pattern, pos + 1, result);
    }
}
```

### 4.2 三种字符的处理

| 字符 | 处理方式 | 为什么这样处理 |
|------|----------|---------------|
| `*` | 两种递归：匹配空串 / 匹配至少一个字符 | 匹配任意长度（0个或多个字符） |
| `?` | 遍历所有子节点，pos+1 | 匹配恰好1个字符 |
| 普通字符 | 精确匹配子节点，pos+1 | 必须完全相同 |

### 4.3 举例解释

**模式 `a*d` 匹配 `"and"`**

```
第1步：'a' 精确匹配 → 到 node_a
第2步：'*' 
    ├── 匹配空串 → 跳过 *，匹配 'd' 
    │   └── node_a 有 'd' 吗？没有 ❌
    └── 匹配至少一个字符 → 进入 'n' 节点，'*' 保留
第3步：继续处理 '*'
    ├── 匹配空串 → 匹配 'd'
    │   └── node_n 有 'd' ✅ → 收集 "and"
    └── 匹配至少一个字符 → 继续深入...
```
