# EOJ-1016 整数幂 - 完整解析

---

## 一、题目理解

### 问题描述

计算 `(A^B) mod C`，其中 A、B、C 都是正整数。

### 数据范围

- A, B ≤ 10000
- C ≤ 1000

### 输入格式

- 多行输入，每行三个整数 A B C
- 当 A = 0, B = 0, C = 0 时结束

### 输出格式

- 每行输出 `(A^B) mod C` 的结果

---

## 二、直接计算的问题

### 问题1：数值巨大

```
A = 10000, B = 10000
A^B = 10000^10000 → 40000 位数！
远超任何数据类型！
```

### 问题2：即使 C 很小，中间值也很大

```
A = 10000, B = 10000, C = 2
A^B 是 40000 位数，但只需要知道 mod 2 的结果
```

### 解决方案：**快速幂取模**

---

## 三、核心数学原理

### 模运算的性质

```
(a × b) % c = ((a % c) × (b % c)) % c
```

### 示例

```
2^3 mod 7
= ((2^2 mod 7) × 2) mod 7
= ((4) × 2) mod 7
= 8 mod 7 = 1
```

### 快速幂思想

```
A^B mod C：
如果 B 是偶数：A^B = (A^(B/2))^2
如果 B 是奇数：A^B = (A^(B/2))^2 × A
```

---

## 四、解题思路

### 快速幂取模算法

```cpp
long long powMod(long long a, long long b, long long c) {
    long long result = 1;
    a = a % c;
    while (b > 0) {
        if (b % 2 == 1) {
            result = (result * a) % c;
        }
        a = (a * a) % c;
        b = b / 2;
    }
    return result;
}
```

### 算法流程

```
输入：a=2, b=3, c=7

初始：result=1, a=2%7=2, b=3

第1次循环 (b=3 > 0)：
  b%2=1 → result = (1×2)%7 = 2
  a = (2×2)%7 = 4
  b = 3/2 = 1

第2次循环 (b=1 > 0)：
  b%2=1 → result = (2×4)%7 = 8%7 = 1
  a = (4×4)%7 = 16%7 = 2
  b = 1/2 = 0

循环结束，返回 result=1
```

## 五、完整代码

```cpp
#include <bits/stdc++.h>
using namespace std;

long long powMod(long long a, long long b, long long c) {
    long long result = 1;
    a = a % c;

    while (b > 0) {
        if (b & 1) {              // b % 2 == 1
            result = (result * a) % c;
        }
        a = (a * a) % c;
        b >>= 1;                  // b = b / 2
    }

    return result;
}

int main() {
    long long A, B, C;

    while (cin >> A >> B >> C) {
        if (A == 0 && B == 0 && C == 0) break;
        cout << powMod(A, B, C) << endl;
    }

    return 0;
}
```

### 1. 头文件

```cpp

```

万能头文件。

### 2. powMod 函数

```cpp
long long powMod(long long a, long long b, long long c) {
    long long result = 1;
    a = a % c;
```

- `result`：累积的结果，初始为 1（乘法的单位元）
- `a = a % c`：先对底数取模，防止后续乘法溢出

---

```cpp
    while (b > 0) {
        if (b & 1) {
            result = (result * a) % c;
        }
        a = (a * a) % c;
        b >>= 1;
    }
```

**逐行解释：**

| 代码 | 含义 |
|------|------|
| `while (b > 0)` | 当指数大于 0 时循环 |
| `if (b & 1)` | 检查 b 的二进制最低位是否为 1（等价于 b % 2 == 1） |
| `result = (result * a) % c` | 如果当前位是 1，乘上 a 并取模 |
| `a = (a * a) % c` | a 平方，对应二进制位的权重 |
| `b >>= 1` | b 右移一位（等价于 b = b / 2） |

## 七、算法执行示例

### 示例1：2 3 7

```
a=2, b=3, c=7
result=1, a=2%7=2

b=3 (二进制 11)
第1次循环：
  b&1=1 → result = 1×2%7 = 2
  a = 2×2%7 = 4
  b = 1

第2次循环：
  b&1=1 → result = 2×4%7 = 8%7 = 1
  a = 4×4%7 = 16%7 = 2
  b = 0

返回 1 ✓
```

---

### 示例2：12 12 10

```
a=12, b=12, c=10

12^12 mod 10 = (12 mod 10)^12 mod 10 = 2^12 mod 10

2^1 mod 10 = 2
2^2 mod 10 = 4
2^3 mod 10 = 8
2^4 mod 10 = 6
2^5 mod 10 = 2
...

2^12 mod 10 = 6 ✓
```

## 八、复杂度分析

| 操作 | 时间复杂度 | 空间复杂度 |
|------|-----------|-----------|
| 快速幂取模 | O(log B) | O(1) |
| 直接计算 | O(B) | O(1) |

**B 最大 10000，但用快速幂只需约 14 次循环！**

## 九、为什么 `b & 1` 能判断奇偶？

```
二进制中，最低位为 1 表示奇数，为 0 表示偶数

5 (101) & 1 = 1 → 奇数
4 (100) & 1 = 0 → 偶数
```

## 十一、总结

| 要点 | 说明 |
|------|------|
| **核心算法** | 快速幂取模 |
| **数学原理** | (a × b) % c = ((a % c) × (b % c)) % c |
| **时间复杂度** | O(log B) |
| **数据范围** | A, B ≤ 10000，long long 足够 |
| **终止条件** | A = 0, B = 0, C = 0 |

**核心思想：利用模运算的性质，在计算过程中不断取模，避免大数溢出；利用二进制分解指数，将 O(B) 降到 O(log B)。**
