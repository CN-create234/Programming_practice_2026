# EOJ-1031 Weights（砝码称重）- 逐行详细解释
## 问题描述

商人使用砝码和两盘天平来称量许多商品。但是，如果使用的砝码种类有限，则只能精确称量某些数量的物品。

例如，假设你只有两个砝码：一个 1 盎司的砝码和一个 3 盎司的砝码。使用它们可以轻松称出 4 盎司，如图1所示。

更有趣的是，你还可以通过将 1 盎司砝码移到另一侧来称出 2 盎司，如图2所示。

编写一个程序，判断是否能用给定的砝码集合称出目标重量。

---

## 输入格式

第一行包含一个整数 \( n \)（\( 1 \leq n \leq 10 \)）—— 给定砝码集合的大小。

第二行包含 \( n \) 个整数 \( w_1, w_2, \ldots, w_n \)（\( 1 \leq w_i \leq 100 \)）—— 每个砝码的重量。

## 输出格式

输出一个二进制字符串 \( s \)，满足：

\[
|s| = \sum_{i=1}^n w_i
\]

\[
s_i = 1 \quad \text{如果可以用给定砝码集合称出目标重量 } W_i = i
\]

\[
s_i = 0 \quad \text{否则}
\]

**注意：** \( s \) 的索引从 1 开始。

---

## 示例

### 输入
```
2
1 3
```

### 输出
```
11111
```

**解释：** 总重量 \( 1 + 3 = 4 \)，所以输出字符串长度为 4。目标重量 1、2、3、4 都可以称出，因此输出 `1111`。

---

## 称重原理

天平有两盘，砝码可以放在：
1. **物品同侧**（相当于减去该砝码重量）
2. **物品对侧**（相当于加上该砝码重量）
3. **不放**（不影响）

对于每个砝码 \( w_i \)，它有三种状态：
- 放在物品对侧：贡献 \( +w_i \)
- 不放：贡献 \( 0 \)
- 放在物品同侧：贡献 \( -w_i \)

目标重量 \( W \) 可称出，当且仅当存在一种砝码分配方式，使得：
\[
\sum_{i=1}^n c_i \times w_i = W
\]
其中 \( c_i \in \{-1, 0, 1\} \)。

## 一、完整代码（集合版本）

```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
    int n;
    cin >> n;

    vector<int> weights(n);
    int total = 0;
    for (int i = 0; i < n; i++) {
        cin >> weights[i];
        total += weights[i];
    }

    set<int> reachable;
    reachable.insert(0);

    for (int w : weights) {
        vector<int> toAdd;
        for (int x : reachable) {
            toAdd.push_back(x + w);
            toAdd.push_back(x - w);
        }
        for (int x : toAdd) {
            reachable.insert(x);
        }
    }

    string ans = "";
    for (int i = 1; i <= total; i++) {
        ans += (reachable.find(i) != reachable.end()) ? '1' : '0';
    }

    cout << ans << endl;

    return 0;
}
```

### 第二部分：读取砝码重量

```cpp
    vector<int> weights(n);
    int total = 0;
    for (int i = 0; i < n; i++) {
        cin >> weights[i];
        total += weights[i];
    }
```

| 行 | 代码 | 含义 |
|----|------|------|
| 1 | `vector<int> weights(n)` | 创建大小为 `n` 的整数向量，存储每个砝码的重量 |
| 2 | `int total = 0` | 初始化总重量为 0 |
| 3 | `for (int i = 0; i < n; i++)` | 循环读取每个砝码 |
| 4 | `cin >> weights[i]` | 读取第 `i` 个砝码的重量 |
| 5 | `total += weights[i]` | 累加总重量 |

**示例：** 输入 `1 3`

```
weights = [1, 3]
total = 4
```

---

### 第三部分：初始化可达重量集合

```cpp
    set<int> reachable;
    reachable.insert(0);
```

| 行 | 代码 | 含义 |
|----|------|------|
| 1 | `set<int> reachable` | 创建一个整数集合 `reachable`，自动去重并排序 |
| 2 | `reachable.insert(0)` | 插入 0（不称任何东西，重量为 0 总是可达的） |

**为什么用 `set`？**
- 自动去重（同一个重量可能有多种组合）
- 自动排序
- 查找效率 O(log n)

**为什么不直接用数组？**
- 重量可能为负数（砝码放同侧时），`set` 可以存储负数
- 总重量 `total` 最大 1000，也可以用数组偏移

---

### 第四部分：逐个砝码扩展

```cpp
    for (int w : weights) {
        vector<int> toAdd;
        for (int x : reachable) {
            toAdd.push_back(x + w);
            toAdd.push_back(x - w);
        }
        for (int x : toAdd) {
            reachable.insert(x);
        }
    }
```

**逐行解释：**

| 行 | 代码 | 含义 |
|----|------|------|
| 1 | `for (int w : weights)` | 遍历每个砝码重量 `w` |
| 2 | `vector<int> toAdd` | 创建临时向量，存储本轮要新增的重量 |
| 3 | `for (int x : reachable)` | 遍历当前所有可达重量 `x` |
| 4 | `toAdd.push_back(x + w)` | 砝码放**物品对侧**：重量增加 `w` |
| 5 | `toAdd.push_back(x - w)` | 砝码放**物品同侧**：重量减少 `w` |
| 6 | `for (int x : toAdd)` | 遍历临时向量 |
| 7 | `reachable.insert(x)` | 将所有新重量加入集合 |

**为什么用 `toAdd` 而不是直接插入 `reachable`？**

```
❌ 错误写法：
for (int x : reachable) {
    reachable.insert(x + w);  // 遍历时插入，迭代器失效！
}

✅ 正确写法：
for (int x : reachable) {
    toAdd.push_back(x + w);   // 先收集
}
for (int x : toAdd) {
    reachable.insert(x);      // 再统一插入
}
```

**为什么不用 `dp` 数组？**
- 重量可能为负数（砝码放同侧），用 `set` 更简单

---

### 第五部分：构造输出字符串

```cpp
    string ans = "";
    for (int i = 1; i <= total; i++) {
        ans += (reachable.find(i) != reachable.end()) ? '1' : '0';
    }
```

| 行 | 代码 | 含义 |
|----|------|------|
| 1 | `string ans = ""` | 初始化结果字符串为空 |
| 2 | `for (int i = 1; i <= total; i++)` | 从 1 到 `total` 遍历每个目标重量 |
| 3 | `reachable.find(i) != reachable.end()` | 判断重量 `i` 是否在可达集合中 |
| 4 | `? '1' : '0'` | 可达输出 `'1'`，否则输出 `'0'` |

**示例：** `reachable = {0, 1, 2, 3, 4}`

```
i=1: find(1) != end → true → '1'
i=2: find(2) != end → true → '1'
i=3: find(3) != end → true → '1'
i=4: find(4) != end → true → '1'
ans = "1111"
```

## 三、完整执行示例

### 输入
```
2
1 3
```

### 执行过程

```
n=2, weights=[1,3], total=4

reachable = {0}

--- 处理 w=1 ---
toAdd = []
遍历 reachable: x=0
  toAdd.push_back(0+1) = 1
  toAdd.push_back(0-1) = -1
toAdd = [-1, 1]
插入后: reachable = {-1, 0, 1}

--- 处理 w=3 ---
toAdd = []
遍历 reachable: x=-1
  toAdd.push_back(-1+3) = 2
  toAdd.push_back(-1-3) = -4
遍历 reachable: x=0
  toAdd.push_back(0+3) = 3
  toAdd.push_back(0-3) = -3
遍历 reachable: x=1
  toAdd.push_back(1+3) = 4
  toAdd.push_back(1-3) = -2
toAdd = [2, -4, 3, -3, 4, -2]
插入后: reachable = {-4, -3, -2, -1, 0, 1, 2, 3, 4}

--- 输出 ---
i=1: 在集合中 → '1'
i=2: 在集合中 → '1'
i=3: 在集合中 → '1'
i=4: 在集合中 → '1'
ans = "1111"
```

**输出：1111** ✓

## 四、DP 版本对比

| 版本 | 优点 | 缺点 |
|------|------|------|
| `set` 版本 | 简单直观，可以处理负数 | 速度稍慢（O(n² log n)） |
| `dp` 版本 | 速度快（O(n × total)） | 需要处理负数偏移 |

本题 `n ≤ 10`，`total ≤ 1000`，两种方法都可以。

---

## 五、总结

| 要点 | 说明 |
|------|------|
| **核心** | 每个砝码有三种状态：+w、0、-w |
| **数据结构** | `set<int>` 存储所有可达重量 |
| **扩展方式** | 每个砝码扩展当前所有可达重量 |
| **避免重复使用** | 用 `toAdd` 先收集再统一插入 |
| **输出** | 从 1 到 total 判断是否可达 |

**核心思想：** 用集合记录所有可达重量，逐个砝码扩展（加、减），最终判断每个目标重量是否可达。
