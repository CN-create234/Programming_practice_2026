## 宏观理解

### 整体思路

```
┌─────────────────────────────────────────────────────────┐
│                    核心策略                              │
├─────────────────────────────────────────────────────────┤
│                                                         │
│   从右向左，尝试"微调"某一位                             │
│                                                         │
│   1. 找到可以增加的一位（从右到左）                     │
│   2. 增加该位                                           │
│   3. 后面的所有位用最小可行数字填充                     │
│   4. 验证是否合法                                       │
│   5. 合法 → 返回，不合法 → 继续尝试更左边的一位         │
│                                                         │
│   6. 所有位都无法增加 → 加一位（如 999 → 1010）        │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

### 为什么这样做？

**目标**：找到 > A 的最小不重复数

**关键观察**：
1. 要得到"最小"的大于 A 的数，应该**尽量少地改变高位**
2. 从右到左尝试，先改**最低位**，因为改低位对数值影响最小
3. 改完一位后，后面的位全部用**最小可行数字**填充

### 类比：找下一个字典序排列

类似于找"下一个排列"的思路：
- 从右到左找可以增加的位置
- 增加后，后面全部取最小值

## 细节解释

### 1. isValid 函数

```cpp
bool isValid(const string& s) {
    for (int i = 1; i < s.size(); i++) {
        if (s[i] == s[i - 1]) return false;
    }
    return true;
}
```

**作用**：检查字符串是否是不重复数

**原理**：遍历每个位置，检查是否和前一个字符相同

| 字符串 | 检查过程 | 结果 |
|--------|---------|------|
| "1234" | 1≠2, 2≠3, 3≠4 | ✅ true |
| "1225" | 1≠2, 2==2 → 返回 false | ❌ false |
| "100" | 1≠0, 0==0 → 返回 false | ❌ false |

**时间复杂度**：O(n)，n 是字符串长度

### 2.1 外层循环：从右到左尝试

```cpp
for (int i = s.size() - 1; i >= 0; i--) {
```

**含义**：从**最低位**（最右边）开始，尝试增加每一位

**为什么从右到左？**
- 改低位对数值影响最小
- 例如：679898 → 修改最后一位 8 得到 679899，比修改高位 6 得到 779898 更小

**策略**：优先修改最低位，不行再往左

---

### 2.2 内层循环：尝试增加当前位

```cpp
for (char d = s[i] + 1; d <= '9'; d++) {
```

**含义**：将第 i 位增加到比原来大的数字

**示例**：
```
s[i] = '5'
d 从 '6' 到 '9' 依次尝试
```

**为什么只尝试增加到 9？**
- 如果增加到 10，就需要进位了
- 进位的情况会由外层循环处理（继续往前尝试）

---

### 2.3 检查是否和前面重复

```cpp
if (i > 0 && d == s[i - 1]) continue;
```

**含义**：如果增加后的数字和前面一位相同，跳过

**示例**：
```
s = "1225"
i = 2, s[2] = '2'
尝试 d = '3'：检查 s[1] = '2'，3 ≠ 2，可以
尝试 d = '2'：但 d 从 s[i]+1 开始，所以不会等于原值
```

---

### 2.4 构建候选答案

```cpp
string ans = s;
ans[i] = d;
```

**含义**：复制原字符串，修改第 i 位为 d

**示例**：
```
s = "679898", i = 5（最后一位）
d = '9'
ans = "679899"
```

---

### 2.5 填充后面的最小数字

```cpp
for (int j = i + 1; j < ans.size(); j++) {
    for (char c = '0'; c <= '9'; c++) {
        if (j > 0 && c == ans[j - 1]) continue;
        ans[j] = c;
        break;
    }
}
```

**含义**：从 i+1 开始，每一位填入**最小可行数字**

**填充策略**：
1. 从 '0' 到 '9' 尝试
2. 选择一个不等于前一位的数字
3. 选择最小的那个

**示例**：
```
ans = "679899"，i = 3，当前 ans[3] = '8'

从 j=4 开始填充：
j=4: ans[3]='8'，尝试 c='0' → 0≠8 ✅，ans[4]='0'
j=5: ans[4]='0'，尝试 c='0' → 0==0 ❌，c='1' → 1≠0 ✅，ans[5]='1'

结果：ans = "679801"
```

**为什么选择最小的？**
- 为了得到**最小**的大于原数的结果
- 在高位确定的情况下，低位越小，整个数越小

---

### 2.6 验证结果

```cpp
if (isValid(ans)) return ans;
```

**含义**：检查填充后的字符串是否是不重复数

**为什么还要验证？**
- 填充过程保证了相邻不重复，但前面可能还有冲突
- 例如：修改了第 i 位后，可能和更前面的位产生冲突
- 验证确保整个字符串合法

**示例**：
```
s = "1225"
i = 2（第3位），d = '3'
ans = "1235"
填充后面：j=3，ans[2]='3'，选 c='0' → ans="1230"
验证：1≠2, 2≠3, 3≠0 ✅ 返回 "1230"
```

### 2.7 所有位都无法增加

```cpp
string ans = "1";
char last = '1';
for (int i = 1; i <= s.size(); i++) {
    for (char d = '0'; d <= '9'; d++) {
        if (d != last) {
            ans += d;
            last = d;
            break;
        }
    }
}
return ans;
```

**触发条件**：所有位都无法增加（如 "999"）

**策略**：加一位，构建最小的不重复数

**构建过程**：
```
s = "999"
ans = "1", last = '1'

i=1: d从'0'开始，0≠1 → ans="10", last='0'
i=2: d从'0'开始，0==0跳过，1≠0 → ans="101", last='1'
i=3: d从'0'开始，0≠1 → ans="1010", last='0'

返回 "1010"
```

**为什么是 "1010" 而不是 "1000"？**
- "1000" 有连续 0，不合法
- 最小的合法数：1, 0, 1, 0 交替

## 算法正确性证明

### 为什么这样能找到最小？

1. **从右到左尝试**：优先改低位，保证结果最小
2. **增加后填充最小数字**：保证在高位确定的情况下最小
3. **验证合法性**：保证结果有效

### 为什么加一位时用 "1010..."？

加一位后的最小不重复数必须：
1. 以 '1' 开头（最高位不能为 0）
2. 每一位不等于前一位
3. 尽量小

所以最优解是：1, 0, 1, 0, ... 交替

## 关键点总结

| 知识点 | 说明 |
|--------|------|
| 从右到左 | 优先改低位，保证结果最小 |
| 尝试增加 | 从 s[i]+1 到 '9' |
| 避免重复 | 增加后和前面不能相同 |
| 填充最小数字 | 从 i+1 开始，每位移最小的可行数字 |
| 验证 | 确保整个字符串合法 |
| 加一位 | 所有位都无法增加时（如 999）|
