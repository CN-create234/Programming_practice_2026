## 一、Trie 数据结构基础

### 第1题
**Trie（字典树）中，每个节点存储的是什么？**

A. 一个完整的单词
B. 一个字符
C. 一个数字
D. 一个句子

<details>
<summary>点击查看答案</summary>

**正确答案：B**

**解析：** Trie 中每个节点代表一个字符，从根节点到某个节点的路径形成一个字符串（前缀）。只有在单词结尾节点才存储完整单词。

```
示例：存储 "and"
root → 'a' → 'n' → 'd'
        ↑      ↑      ↑
      字符    字符    字符（单词结尾）
```
</details>

---

### 第2题
**在 Trie 中，`isWord` 标记的作用是什么？**

A. 标记节点是否有子节点
B. 标记从根到该节点的路径是否构成一个完整单词
C. 标记节点是否被删除
D. 标记节点的深度

<details>
<summary>点击查看答案</summary>

**正确答案：B**

**解析：** `isWord = true` 表示从根节点到当前节点的路径是一个完整的单词。例如路径 `root → a → n → d`，在 'd' 节点 `isWord = true`，表示 "and" 是一个完整单词。

```
示例：
root → a → n → d (isWord=true)  → "and"
              → o (isWord=true)  → "ando"（共享 "an" 前缀）
```
</details>

---

### 第3题
**以下哪个是 Trie 的主要优势？**

A. 占用内存最少
B. 按字典序自动排序
C. 共享前缀，节省空间，查询效率高
D. 支持数字运算

<details>
<summary>点击查看答案</summary>

**正确答案：C**

**解析：** Trie 的核心优势是**共享前缀**。例如 "and"、"ando"、"another" 共享前缀 "an"，不需要重复存储。

```
共享前缀示例：
        root
         |
         a
         |
         n
        / \
       d   o
      /     \
    "and"  "ando"
```
</details>

## 三、排序规则

### 第7题
**词库中有以下单词：`apple(5)`, `and(3)`, `zoo(5)`, `about(3)`，按排序规则排列，第一个是？**

A. apple
B. and
C. zoo
D. about

<details>
<summary>点击查看答案</summary>

**正确答案：D**

**解析：** 排序规则：**词频降序 → 字典序升序**

```
词频 5: apple, zoo → 按字典序: apple, zoo
词频 3: about, and → 按字典序: about, and

完整排序:
1. apple (5)    ← 词频最高
2. zoo (5)      ← 词频最高
3. about (3)    ← 词频次高，字典序 a 开头
4. and (3)      ← 词频次高，字典序 a 开头
```
</details>

---

## 五、缓存机制

### 第12题
**以下哪个操作会清空 Top-K 缓存？**

A. `freq and`
B. `list`
C. `select 1`
D. `prefix an`

<details>
<summary>点击查看答案</summary>

**正确答案：C**

**解析：** `select` 会修改词频（+1），词频变化影响排序结果，因此必须清空相关缓存。

触发缓存清空的操作：
- `add` → 添加新词
- `delete` → 删除单词
- `update` → 修改词频
- `select` → 词频 +1

不触发缓存清空的操作：
- `freq` → 只读查询
- `list` → 只读查询
- `prefix` → 只读查询
</details>

---

### 第13题
**为什么需要 Top-K 缓存？**

A. 减少内存使用
B. 加快重复查询速度
C. 增加排序准确性
D. 支持更多单词

<details>
<summary>点击查看答案</summary>

**正确答案：B**

**解析：** 用户可能多次输入相同的数字序列，每次从 Trie 查询都要遍历和排序，耗时。缓存可以 O(1) 返回结果。

```
第一次 input 26 → 查询 Trie + 排序 → 耗时 O(3^L + M log M)
第二次 input 26 → 直接返回缓存 → 耗时 O(1)  ⚡
```
</details>

## 六、通配符匹配

### 第14题
**模式 `a?d` 可以匹配以下哪个单词？**

A. ad
B. and
C. a
D. abcd

<details>
<summary>点击查看答案</summary>

**正确答案：B**

**解析：** `?` 匹配任意**单个**字符。
- `a?d` = `a` + 任意1个字符 + `d`
- "and" = a + n + d ✓
- "ad" = 只有2个字符 ✗
- "abcd" = a + bc + d（中间2个字符）✗
</details>

### 第15题
**模式 `a*d` 可以匹配以下哪些单词？**

A. ad
B. and
C. abcd
D. 以上全部

<details>
<summary>点击查看答案</summary>

**正确答案：D**

**解析：** `*` 匹配任意**长度**字符串（包括空串）。
- `a*d` = `a` + 任意长度字符串 + `d`
- "ad" = a + "" + d ✓
- "and" = a + "n" + d ✓
- "abcd" = a + "bc" + d ✓
</details>

A. 使用 AI 技术

D. 语音识别

<details>
<summary>点击查看答案</summary>

**正确答案：B**

**解析：** 系统的"智能"体现在**词频学习**。用户选择某个单词后，该单词词频 +1，下次查询时排序会变化（高频词排前面），形成"输入→选择→学习→下次更准"的闭环。
</details>

### 第22题
**以下哪个是系统的完整交互闭环？**

A. input → select → input → select
B. input → select → update → list
C. add → input → list
D. select → input → select

<details>
<summary>点击查看答案</summary>

**正确答案：A**

**解析：** 完整闭环是 `input（展示候选）→ select（选择）→ input（再次查询，体现变化）`。每次 `select` 改变词频，下次 `input` 反映变化。
</details>

### 第23题
**为什么删除单词时只取消 `isWord` 标记，不删除节点？**

A. 因为删除节点太慢
B. 因为其他单词可能共享前缀
C. 因为可以恢复
D. 因为内存不足

<details>
<summary>点击查看答案</summary>

**正确答案：B**

**解析：** 多个单词可能共享前缀。例如 "and" 和 "ando" 共享 "and"，删除 "and" 时不能删除节点，否则 "ando" 的路径会被破坏。

```
删除 "and" 前：
root → a → n → d (and)
              → o (ando)

删除 "and" 后（只取消标记）：
root → a → n → d (不是单词)
              → o (ando)  ← 仍然可以访问！
```
</details>

</details>
