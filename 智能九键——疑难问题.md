# `unique_ptr` 智能指针详解

## 一、什么是 `unique_ptr`？

`unique_ptr` 是 C++11 引入的**智能指针**，用于**自动管理动态分配的内存**。

```cpp
#include <memory>  // 使用 unique_ptr 需要包含这个头文件

unique_ptr<TrieNode> ptr = make_unique<TrieNode>();
// ↑ 自动分配内存
// ↑ 当 ptr 离开作用域时，自动释放内存
```

---

## 二、为什么需要智能指针？

### 传统指针的问题

```cpp
// ❌ 传统指针 - 容易出错
TrieNode* node = new TrieNode();  // 手动分配内存
// ... 使用 node ...
delete node;  // 必须手动释放！如果忘记删除 → 内存泄漏
```

**常见问题**：
1. **忘记 `delete`** → 内存泄漏
2. **多次 `delete`** → 程序崩溃
3. **异常发生时** → `delete` 不会执行 → 内存泄漏

### 智能指针的优势

```cpp
// ✅ unique_ptr - 自动管理
unique_ptr<TrieNode> node = make_unique<TrieNode>();
// ... 使用 node ...
// 不需要 delete！离开作用域时自动释放
```

---

## 三、`unique_ptr` 的核心特性

### 1. 独占所有权

```cpp
unique_ptr<TrieNode> ptr1 = make_unique<TrieNode>();
unique_ptr<TrieNode> ptr2 = ptr1;  // ❌ 编译错误！
// unique_ptr 不能被拷贝，只能被移动
```

**为什么独占？**
- 每个资源只能有一个所有者
- 避免多个指针指向同一块内存导致重复释放

### 2. 移动语义

```cpp
unique_ptr<TrieNode> ptr1 = make_unique<TrieNode>();
unique_ptr<TrieNode> ptr2 = std::move(ptr1);  // ✅ 所有权转移
// 现在 ptr1 为空，ptr2 拥有资源
```

### 3. 自动释放

```cpp
void function() {
    auto ptr = make_unique<TrieNode>();
    // 离开函数时，ptr 自动调用析构函数释放内存
}
```

## 四、在 Trie 代码中的使用

### 4.1 节点中的子节点存储

```cpp
struct TrieNode {
    // 子节点映射：字符 → 子节点
    // 每个子节点都使用 unique_ptr 管理
    unordered_map<char, unique_ptr<TrieNode>> children;
    // ...
};
```

**为什么在这里用 `unique_ptr`？**

```
Trie 树结构：
        [root]
        /    \
       a      t
      / \      \
     n   n      a
    /     \      \
   d       o      n
  /         \      \
 and        ando    d
                    |
                   tand

每个节点只能有一个父节点
→ 所有权唯一 → 适合 unique_ptr
```

### 4.2 创建子节点

```cpp
// 在 addWord 中
if (!node->children.count(ch)) {
    // 创建新节点，自动管理内存
    node->children[ch] = make_unique<TrieNode>();
}
// ↑ 相当于 new TrieNode()，但不需要 delete
```

### 4.3 获取原始指针

```cpp
// 通过 .get() 获取原始指针，用于遍历
TrieNode* node = root_.get();
// ↑ 只是为了操作，不转移所有权
```

## 五、`make_unique` 详解

```cpp
// make_unique<T>() 是创建 unique_ptr 的推荐方式
auto ptr = make_unique<TrieNode>();  // C++14 起

// 等价于：
unique_ptr<TrieNode> ptr(new TrieNode());

// 但 make_unique 更安全（避免异常时的内存泄漏）
```

### 为什么用 `make_unique` 而不是 `new`？

```cpp
// ❌ 不推荐：直接 new
unique_ptr<TrieNode> ptr(new TrieNode());

// ✅ 推荐：make_unique
auto ptr = make_unique<TrieNode>();

// 好处：
// 1. 代码更简洁
// 2. 异常安全
// 3. 与 make_shared 风格一致
```

### 6.1 成员变量

```cpp
class Trie {

```

### 6.2 构造函数

```cpp
Trie::Trie() : root_(make_unique<TrieNode>()) {}
// ↑ 创建根节点，自动管理生命周期
```

### 6.3 添加单词时创建节点

```cpp
bool Trie::addWord(const string& word, int freq) {
    TrieNode* node = root_.get();  // 获取原始指针
    
    for (char ch : word) {
        if (!node->children.count(ch)) {
            // 创建新的子节点
            node->children[ch] = make_unique<TrieNode>();
        }
        node = node->children[ch].get();  // 移动到子节点
    }
    // ...
}
```

### 6.4 遍历子节点

```cpp
void collectWords(TrieNode* node, vector<pair<string, int>>& result) const {
    if (node->isWord) {
        result.emplace_back(node->word, node->freq);
    }
    
    // 遍历所有子节点
    // auto& [ch, child] 是 C++17 的结构化绑定
    // child 的类型是 unique_ptr<TrieNode>&
    for (auto& [ch, child] : node->children) {
        // child.get() 获取原始指针传递给递归函数
        collectWords(child.get(), result);
    }
}
```

## 七、内存管理对比

### 传统指针（需要手动管理）

```cpp
// ❌ 传统方式 - 容易出错
class Trie {
private:
    TrieNode* root_;
public:
    Trie() {
        root_ = new TrieNode();  // 分配
    }
    ~Trie() {
        delete root_;  // 必须手动释放！
        // 还要递归删除所有子节点！
    }
};
```

### unique_ptr（自动管理）

```cpp
// ✅ 现代方式 - 自动管理
class Trie {
private:
    unique_ptr<TrieNode> root_;
public:
    Trie() : root_(make_unique<TrieNode>()) {}
    // 不需要析构函数！
    // root_ 离开作用域时自动释放所有子节点
};
```

## 八、`unique_ptr` 的常用操作

| 操作 | 代码 | 说明 |
|------|------|------|
| 创建 | `auto p = make_unique<T>()` | 创建智能指针 |
| 获取原始指针 | `p.get()` | 用于函数调用 |
| 解引用 | `*p` 或 `p->` | 访问对象成员 |
| 释放所有权 | `p.release()` | 返回原始指针，不再管理 |
| 重置 | `p.reset()` | 释放并重置为空 |
| 判断是否为空 | `if (p)` | 检查是否有效 |

---

## 九、为什么不用其他智能指针？

### 对比表

| 智能指针 | 所有权 | 引用计数 | 适用场景 |
|---------|--------|---------|---------|
| `unique_ptr` | 独占 | 无 | 树、链表等独占结构 |
| `shared_ptr` | 共享 | 有 | 多个对象共享同一资源 |
| `weak_ptr` | 弱引用 | 无 | 解决循环引用 |

### Trie 中的选择

```
Trie 树的特点：
1. 每个节点只有一个父节点（根节点除外）
2. 父节点拥有子节点
3. 不需要共享所有权

→ 最适合 unique_ptr（独占所有权）
```

## 十、内存释放过程

```
当 Trie 对象销毁时：

Trie 析构
    ↓
root_ 析构 (unique_ptr)
    ↓
调用 TrieNode 析构函数
    ↓
children 析构 (unordered_map)
    ↓
遍历所有子节点，逐个调用 unique_ptr 析构
    ↓
递归释放所有子节点
    ↓
所有内存自动释放！
```

**整个过程自动完成，不需要手动 `delete`！**

## 三、命令处理函数 - `cmdInput()`

```cpp
/**
 * system.cpp - cmdInput() 函数详解
 * 
 * 功能：处理用户的 input 命令
 * 格式：input <数字序列>
 */
void NineKeySystem::cmdInput(const vector<string>& args) {
    // ============================================================
    // 步骤1: 检查参数数量
    // ============================================================
    // input 命令需要恰好 2 个参数：命令名 + 数字序列
    // 例：input 26 → args = ["input", "26"]
    if (args.size() != 2) {
        cout << "用法: input <数字序列>" << endl;
        return;
    }

    // ============================================================
    // 步骤2: 验证数字序列
    // ============================================================
    // 数字序列只能包含 2-9（任务书要求）
    // 不能包含 0、1 或其他字符
    if (!isValidDigits(args[1])) {
        cout << "错误: 数字序列只能包含 2-9" << endl;
        invalidateCache();  // 清空候选列表
        return;
    }

    // ============================================================
    // 步骤3: 调用 Trie 查询
    // ============================================================
    // digitQuery() 返回匹配的候选列表（已排序、已取 Top-K）
    lastCandidates_ = trie_.digitQuery(args[1]);
    
    // 判断是否有候选词
    hasCandidates_ = !lastCandidates_.empty();

    // ============================================================
    // 步骤4: 展示结果
    // ============================================================
    if (lastCandidates_.empty()) {
        // 无匹配单词，输出 "no word"（任务书要求）
        cout << "无匹配单词" << endl;
    } else {
        // 按序号展示候选列表
        // 格式: 序号. 单词 词频
        for (size_t i = 0; i < lastCandidates_.size(); ++i) {
            cout << (i + 1) << ". " << lastCandidates_[i].first
                 << " " << lastCandidates_[i].second << endl;
        }
    }
}
```

## 四、核心查询函数 - `digitQuery()`

```cpp
/**
 * trie.cpp - digitQuery() 详解
 * 
 * 功能：根据数字序列查询匹配的单词（带缓存）
 * 返回：Top-K 个候选词
 */
vector<pair<string, int>> Trie::digitQuery(const string& digits) const {
    // ============================================================
    // 步骤1: 检查缓存
    // ============================================================
    // 如果之前查询过相同的数字序列，直接返回缓存结果
    // 大幅提升重复查询的速度
    auto it = cache_.find(digits);
    if (it != cache_.end()) {
        return it->second;  // 缓存命中！
    }

    // ============================================================
    // 步骤2: 缓存未命中，执行查询
    // ============================================================
    vector<pair<string, int>> result;
    
    // collectByDigit() 是递归函数
    // 参数说明：
    //   - root_.get(): 从根节点开始
    //   - "": 当前已匹配的字符串（初始为空）
    //   - digits: 完整的数字序列 "26"
    //   - 0: 当前处理的数字位置
    //   - result: 收集结果的引用
    collectByDigit(root_.get(), "", digits, 0, result);

    // ============================================================
    // 步骤3: 排序并取 Top-K
    // ============================================================
    // 排序规则：词频降序 → 字典序升序
    sortByFreqAndLex(result);
    
    // 只保留前 K 个候选（K = 5）
    if (result.size() > K) {
        result.resize(K);
    }

    // ============================================================
    // 步骤4: 存入缓存
    // ============================================================
    // const_cast 去掉 const 限制，以便修改缓存
    const_cast<Trie*>(this)->cache_[digits] = result;

    return result;
}
```

## 五、数字匹配递归 - `collectByDigit()`

这是整个系统的**核心算法**！

### 5.1 函数详解

```cpp
/**
 * collectByDigit() - 数字匹配递归函数
 * 
 * 原理：每个数字对应多个字母，递归展开所有可能性
 * 
 * 例如：digits = "26"
 * 
 * 数字 '2' → 字母 'a', 'b', 'c'
 * 数字 '6' → 字母 'm', 'n', 'o'
 * 
 * 所有组合：
 *   a + m = "am"
 *   a + n = "an"  ← 匹配 and, ando, another
 *   a + o = "ao"
 *   b + m = "bm"
 *   b + n = "bn"
 *   b + o = "bo"
 *   c + m = "cm"
 *   c + n = "cn"
 *   c + o = "co"
 * 
 * 然后在 Trie 中查找这些组合对应的单词
 */
void Trie::collectByDigit(TrieNode* node, const string& current,
                          const string& digits, size_t pos,
                          vector<pair<string, int>>& result) const {
    // ============================================================
    // 递归终止条件：处理完所有数字
    // ============================================================
    // 当 pos == digits.length() 时，说明已经匹配完所有数字
    // 此时 current 就是匹配到的前缀
    // 从当前节点收集所有完整单词
    if (pos == digits.length()) {
        collectWords(node, result);
        return;
    }

    // ============================================================
    // 获取当前数字对应的字母集合
    // ============================================================
    char digit = digits[pos];
    auto it = KEY_MAP.find(digit);
    if (it == KEY_MAP.end()) return;  // 非法数字

    const string& letters = it->second;

    // ============================================================
    // 遍历每个可能的字母，递归匹配
    // ============================================================
    // 例如：digit='2', letters="abc"
    // 分别尝试 'a', 'b', 'c'
    for (char ch : letters) {
        // 如果 Trie 中存在该字母对应的子节点
        if (node->children.count(ch)) {
            // 递归匹配下一个数字
            // current + ch: 更新已匹配的字符串
            // pos + 1: 处理下一个数字
            collectByDigit(node->children[ch].get(), current + ch,
                           digits, pos + 1, result);
        }
    }
}
```

## 九、关键点总结

| 要点 | 说明 |
|------|------|
| **命令格式** | `input <数字序列>` |
| **数字范围** | 只能包含 2-9 |
| **查询原理** | 数字→字母展开，递归匹配 Trie |
| **排序规则** | 词频降序 → 字典序升序 |
| **Top-K** | 只返回前 5 个候选 |
| **缓存机制** | 相同数字序列二次查询 O(1) |
| **状态管理** | 保存到 `lastCandidates_` 供 `select` 使用 |
| **无匹配** | 输出 "无匹配单词" |
