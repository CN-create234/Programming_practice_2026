## 逐段详细解释

### 第一部分：GCD 函数

```cpp
long long gcd_val(long long a, long long b) {
    return b == 0 ? a : gcd_val(b, a % b);
}
```

**作用**：计算两个数的最大公约数（GCD）

**原理**：欧几里得算法
```
gcd(a, b) = gcd(b, a % b)
当 b = 0 时，gcd(a, 0) = a
```

**示例**：
```
gcd(12, 8) → gcd(8, 4) → gcd(4, 0) → 4
```

**为什么需要**：用于约分分数（如 4/6 → 2/3）

### 第三部分：读取数据

```cpp

```

**含义**：
- `W`：宽度（位置个数）
- `V`：总下雨量
- `if (!(cin >> W >> V)) return 0;`：如果读取失败，直接退出

```cpp
vector<long long> H(W);

for (int i = 0; i < W; ++i) {
    cin >> H[i];
    min_H = min(min_H, H[i]);

```

**含义**：
- 读取 W 个高度值
- `min_H = 2e18`：初始化为一个非常大的数（2×10^18）
- 每读一个高度，更新最小值

**2e18 是什么意思？**
```
2e18 = 2 × 10^18 = 2000000000000000000
```
这是一个远大于任何可能高度值的数（高度 ≤ 1000），用作初始最小值。

### 第四部分：二分查找水位

```cpp
double low = min_H;
double high = min_H + V + 1.0;
```

**含义**：
- `low`：水位的下限（至少是最低高度）
- `high`：水位的上限（最低高度 + 总水量 + 1）

**为什么 high = min_H + V + 1.0？**
- 最坏情况：所有位置高度都是 min_H
- 水位上升 1 单位需要 W 单位水量
- 总水量 V 最多使水位上升 V/W（≤ V）
- 所以高水位一定 ≤ min_H + V
- 加 1 是为了安全（保证 high 足够大）

```cpp
for (int iter = 0; iter < 100; ++iter) {
    double mid = low + (high - low) / 2.0;
    double current_water = 0;
    for (int i = 0; i < W; ++i) {
        if (mid > H[i]) current_water += (mid - H[i]);
    }
    if (current_water < V) low = mid;
    else high = mid;
}
```

**二分法过程**：

| 步骤 | 说明 |
|------|------|
| 1. 取中间水位 `mid` | `mid = (low + high) / 2` |
| 2. 计算储水量 | 对每个位置，如果 `mid > 高度`，储水 = mid - 高度 |
| 3. 判断 | 如果储水量 < V，水位太低 → `low = mid` |
| 4. 否则 | 水位太高 → `high = mid` |
| 5. 重复 100 次 | 得到高精度结果 |

**为什么循环 100 次？**
- 每次循环区间缩小一半
- 100 次后精度约 2^-100，远超过 double 精度
- 保证结果足够精确

### 第五部分：将浮点数转为分数

```cpp
double final_L = low;
```

保存二分法得到的水位值。

```cpp
long long best_num = 0;
long long best_den = 1;
double min_diff = 1e9;
```

**变量含义**：
| 变量 | 含义 |
|------|------|
| `best_num` | 最佳分数的分子 |
| `best_den` | 最佳分数的分母 |
| `min_diff` | 最小误差（初始极大） |

```cpp

    long long n = round(final_L * d);
    double val = (double)n / d;
```

**作用**：遍历所有可能的分母（1 到 W），找到最接近 `final_L` 的分数。

**为什么分母范围是 1 到 W？**
- 水位 = min_H + V / 宽度
- 分数的分母 = 宽度（或宽度的因数）
- 宽度 ≤ W，所以枚举 1 到 W 足够

**示例**：
```
final_L = 2.666666...
W = 3

d=1: n=round(2.666)=3, val=3, diff=0.333
d=2: n=round(5.333)=5, val=2.5, diff=0.167
d=3: n=round(8)=8, val=2.666..., diff=0
选择 d=3, n=8 → 8/3 = 2+2/3
```
