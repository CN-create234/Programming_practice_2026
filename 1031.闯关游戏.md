## 一、问题本质

题目描述了一个场景：有 `N` 个怪兽，每个怪兽需要花费 `t_i` 分钟来击败，击败后获得 `p_i` 奖励分。你总共有 `T` 分钟，可以选择任意怪兽、任意顺序，但每个怪兽只能打一次（被打败就没了）。目标是**在总时间不超过 `T` 的前提下，最大化总奖励分**。

这是一个典型的 **0/1 背包问题（0/1 Knapsack Problem）**：
- **物品**：每个怪兽
- **重量**：花费时间 `t_i`
- **价值**：奖励分 `p_i`
- **背包容量**：总时间 `T`
- **目标**：在容量限制下，最大化总价值

---

## 二、数据范围分析

根据题目给出的数据范围：

- `1 ≤ N ≤ 100`：怪兽数量最多 100 个，很小。
- `1 ≤ T ≤ 10^4`：总时间最多 10000，背包容量中等。
- `1 ≤ t_i ≤ 10^4`：每个怪兽耗时也在 10000 以内。
- `1 ≤ p_i ≤ 10^50`：奖励分数可以是一个**最多 50 位的整数**。

这意味着：
- 我们可以用 `dp[T+1]` 数组，大小约 10001，完全可行。
- 但 `p_i` 超出了 C++ 任何内置整数类型的范围：
  - `long long`：最大约 9.22×10^18（19位）
  - `__int128`：最大约 1.7×10^38（39位）
  - `10^50` 有 51 位，远超它们。

因此，**必须使用高精度整数**来处理奖励值的存储、加法和比较。

---

## 三、动态规划（DP）设计

### 状态定义
`dp[j]` 表示：在**恰好或不超过** `j` 分钟的情况下，能获得的最大奖励分。

通常我们定义为“不超过 j 分钟”，这样初始化全 0，最后直接输出 `dp[T]` 即可。

### 状态转移
对于每个怪兽 `(t, v)`，我们考虑**是否选择它**：

如果不选：`dp[j]` 保持不变  
如果选：`dp[j] = max(dp[j], dp[j - t] + v)`

因为每个怪兽只能选一次，所以我们要**逆序**更新 `j`（从 `T` 到 `t`），避免重复选择同一个怪兽。

### 时间复杂度
- 外层循环：N 次（最多 100）
- 内层循环：T 次（最多 10000）
- 每次操作涉及高精度加法和比较（长度最多 50 位）
- 总复杂度：O(N × T × L)，L 是大数位数（≤50），完全可行。

## 四、高精度实现细节

我们需要支持两个操作：

### 1. 大数加法：`add(a, b)`
- 输入：两个字符串表示的非负整数（无前导零）
- 输出：它们的和（字符串形式）
- 实现：从低位到高位逐位相加，处理进位。

### 2. 大数比较：`greaterThan(a, b)`
- 判断 `a > b` 是否成立
- 先比较长度，长度大的数更大；长度相同则按字典序比较。

注意：
- 所有奖励值都是正整数，所以没有负数情况。
- 我们只需要加法和比较，不需要减法或乘法。

## 六、状态转移代码详解

```cpp
for (int i = 0; i < n; i++) {
    int t;
    string v;
    cin >> t >> v;

    for (int j = T; j >= t; j--) {
        // 候选值：如果选当前怪兽，则之前时间 j-t 的最大奖励加上当前怪兽的奖励
        string candidate = add(dp[j - t], v);
        
        // 如果候选值比当前 dp[j] 更大，则更新
        if (greaterThan(candidate, dp[j])) {
            dp[j] = candidate;
        }
    }
}
```

**为什么逆序？**  
- 如果正序更新（从小到大），那么同一个怪兽可能会被多次使用，因为更新后的 `dp[j]` 可能会被后续更大的 `j` 再次利用。  
- 逆序则保证每个怪兽只被考虑一次，符合 0/1 背包的要求。

## 七、完整代码（含注释）

```cpp
#include <iostream>
#include <vector>
#include <string>
#include <algorithm>
using namespace std;

// 高精度加法：返回 a + b
string add(string a, string b) {
    int i = a.size() - 1;      // a 的末位索引
    int j = b.size() - 1;      // b 的末位索引
    int carry = 0;             // 进位
    string res = "";

    while (i >= 0 || j >= 0 || carry) {
        int sum = carry;
        if (i >= 0) sum += a[i--] - '0';
        if (j >= 0) sum += b[j--] - '0';
        res.push_back((sum % 10) + '0');
        carry = sum / 10;
    }

    reverse(res.begin(), res.end());
    return res;
}

// 比较两个大数：如果 a > b 返回 true
bool greaterThan(string a, string b) {
    if (a.size() != b.size())
        return a.size() > b.size();
    return a > b;
}

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int n;
    int T;
    cin >> n >> T;

    // dp[j]：在 j 分钟内能获得的最大奖励（用字符串存储大数）
    vector<string> dp(T + 1, "0");

    for (int i = 0; i < n; i++) {
        int t;
        string v;
        cin >> t >> v;

        // 0/1 背包，逆序更新
        for (int j = T; j >= t; j--) {
            string candidate = add(dp[j - t], v);
            if (greaterThan(candidate, dp[j])) {
                dp[j] = candidate;
            }
        }
    }

    cout << dp[T] << endl;
    return 0;
}
```

## 九、为什么可以用字符串直接比较？

因为所有数字都是正整数，且无前导零，所以：
- 字符串长度更长的数字更大
- 长度相同，字典序比较就是数值比较（如 "999" > "100"）

所以我们直接用字符串比较是安全的。

### 2. `add` 函数（高精度加法）

```cpp
string add(string a, string b) {
```
- 定义一个函数，接收两个字符串 `a` 和 `b`，返回它们的和（也是字符串）。
- 这两个字符串都是非负整数，无前导零。

```cpp
    int i = a.size() - 1;
    int j = b.size() - 1;
```
- `i` 指向 `a` 的最后一位（个位）。
- `j` 指向 `b` 的最后一位（个位）。
- 例如 `a = "123"`，则 `i = 2`，指向字符 `'3'`。

```cpp

```
- `carry` 表示进位，初始为 0。

```cpp
    string res = "";
```
- `res` 用来存储结果，初始为空字符串。

```cpp

```
- 循环条件：只要 `a` 还有位没处理，或 `b` 还有位没处理，或还有进位，就继续。
- 比如 `999 + 1` 时，处理完所有位后还有进位 `1`，需要继续处理。

```cpp
        int sum = carry;
```
- `sum` 初始为进位值。

```cpp

```
- 如果 `a` 还有位，把 `a[i]` 转成数字（`'3' - '0' = 3`），加到 `sum` 中。
- `i--` 表示取完当前位后，指针向左移动一位。

```cpp

```
- 同理，处理 `b` 的当前位。

```cpp
        res.push_back((sum % 10) + '0');
```
- `sum % 10` 得到当前位的数字（0~9），转成字符后追加到 `res` 末尾。
- 注意：这里是从低位到高位存储（个位先存）。

```cpp
        carry = sum / 10;
```
- 更新进位，`sum / 10` 是十位以上的部分。

```cpp
    }
```
- 循环结束，所有位都处理完了。

```cpp
    reverse(res.begin(), res.end());
```
- 因为 `res` 是低位在前（如 `"321"` 表示 123），需要反转成正常顺序 `"123"`。

```cpp

```
