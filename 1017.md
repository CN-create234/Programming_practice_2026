# 1017 负二进制转换 - 逐行详细解释

---

## 一、完整代码

```cpp
#include <bits/stdc++.h>
#include <boost/multiprecision/cpp_int.hpp>
using namespace std;
using namespace boost::multiprecision;

int main() {
    string s;
    cin >> s;
    cpp_int n(s);

    if (n == 0) {
        cout << "0\n";
        return 0;
    }

    string res;
    while (n != 0) {
        cpp_int rem_cpp = n % -2;
        int rem = rem_cpp.convert_to<int>();
        n /= -2;
        if (rem < 0) {
            rem += 2;
            n += 1;
        }
        res.push_back(char('0' + rem));
    }

    reverse(res.begin(), res.end());
    cout << res << '\n';
    return 0;
}
```

### 第二部分：大整数初始化

```cpp
    cpp_int n(s);
```

| 代码 | 含义 |
|------|------|
| `cpp_int n(s)` | 使用 Boost 库的 `cpp_int` 类型，从字符串 `s` 构造大整数 `n` |

**`cpp_int` 是什么？**

`cpp_int` 是 Boost 库（`boost/multiprecision/cpp_int.hpp`）提供的**任意精度整数**类型，可以存储任意大小的整数，不受 `long long` 的 64 位限制。

**为什么用 `cpp_int`？**

题目中十进制整数范围是 `-10^100` 到 `10^100`，远超 `long long` 的 `9.22×10^18`，必须用大整数类型。

**注意：** 需要包含头文件 `#include <boost/multiprecision/cpp_int.hpp>`，但 `bits/stdc++.h` 不包含 Boost 库，所以实际使用需要额外包含。

**完整包含：**
```cpp
#include <bits/stdc++.h>
#include <boost/multiprecision/cpp_int.hpp>
using namespace std;
using namespace boost::multiprecision;
```

### 第六部分：取余数

```cpp
        cpp_int rem_cpp = n % -2;
        int rem = rem_cpp.convert_to<int>();
```

| 行 | 代码 | 含义 |
|----|------|------|
| 1 | `cpp_int rem_cpp = n % -2` | 计算 `n` 除以 `-2` 的余数，结果仍是 `cpp_int` 类型 |
| 2 | `int rem = rem_cpp.convert_to<int>()` | 将余数转换为 `int` 类型 |

**关键问题：** `n % -2` 的结果可能是什么？

在 C++ 中，负数的取余结果符号与被除数相同。

**示例：**
```
-3 % -2 = -1
2 % -2 = 0
-1 % -2 = -1
1 % -2 = 1
```

所以 `rem` 可能是 `-1`、`0` 或 `1`。

**但负二进制的数码只能是 `0` 或 `1`！** 所以需要调整。

---

### 第七部分：求商

```cpp
        n /= -2;
```

| 代码 | 含义 |
|------|------|
| `n /= -2` | `n` 除以 `-2`，商存入 `n` |

**注意：** C++ 的整数除法向零取整。

**示例：**
```
-3 / -2 = 1  （向零取整，1.5 → 1）
-1 / -2 = 0  （0.5 → 0）
1 / -2 = 0   （-0.5 → 0）
```

---

### 第八部分：调整余数（核心！）

```cpp
        if (rem < 0) {
            rem += 2;
            n += 1;
        }
```

| 行 | 代码 | 含义 |
|----|------|------|
| 1 | `if (rem < 0)` | 如果余数是负数（-1） |
| 2 | `rem += 2` | 余数加 2，调整为 1 |
| 3 | `n += 1` | 商加 1，抵消调整 |

**为什么需要调整？**

负二进制的数码只能是 `0` 或 `1`，但 `n % -2` 可能产生 `-1`，需要调整为 `1`。

**数学原理：**

在模运算中，`-1` 和 `1` 在模 2 意义下是等价的：
```
-1 ≡ 1 (mod 2)
```

调整余数后，需要相应调整商，保持等式成立。

**推导：**
```
原等式：n = (-2) × q + r
其中 r = -1

调整后：r' = 1 = r + 2
         n = (-2) × q + (r' - 2)
           = (-2) × q + r' - 2
           = (-2) × (q + 1) + r'
所以新的商 q' = q + 1
```

---

### 第九部分：记录数码

```cpp
        res.push_back(char('0' + rem));
```

| 代码 | 含义 |
|------|------|
| `char('0' + rem)` | 将数字 `rem`（0 或 1）转为字符 `'0'` 或 `'1'` |
| `res.push_back(...)` | 将字符追加到结果字符串末尾 |

**示例：**
```
rem = 0 → '0' + 0 = '0'
rem = 1 → '0' + 1 = '1'
```

---

### 第十部分：反转和输出

```cpp
    reverse(res.begin(), res.end());
```

| 代码 | 含义 |
|------|------|
| `reverse(res.begin(), res.end())` | 反转结果字符串 |

**为什么需要反转？**

除基取余法得到的余数顺序是从**低位到高位**，需要反转才能得到从高位到低位的正确顺序。

**示例：** -3 的转换

```
第1次：rem=1 → res="1"
第2次：rem=0 → res="10"
第3次：rem=1 → res="101"
第4次：rem=1 → res="1011"
反转后：res="1101" ✓
```

## 三、完整执行示例

### 输入：`-3`

```
s = "-3"
n = -3

--- 第1次循环 ---
n = -3, n != 0
rem_cpp = -3 % -2 = -1
rem = -1
n = -3 / -2 = 1
rem < 0 → rem = -1 + 2 = 1, n = 1 + 1 = 2
res.push_back('1') → res = "1"

--- 第2次循环 ---
n = 2, n != 0
rem_cpp = 2 % -2 = 0
rem = 0
n = 2 / -2 = -1
rem >= 0 → 不调整
res.push_back('0') → res = "10"

--- 第3次循环 ---
n = -1, n != 0
rem_cpp = -1 % -2 = -1
rem = -1
n = -1 / -2 = 0
rem < 0 → rem = -1 + 2 = 1, n = 0 + 1 = 1
res.push_back('1') → res = "101"

--- 第4次循环 ---
n = 1, n != 0
rem_cpp = 1 % -2 = 1
rem = 1
n = 1 / -2 = 0
rem >= 0 → 不调整
res.push_back('1') → res = "1011"

--- 循环结束 ---
n = 0
reverse(res) → "1101"
输出：1101 ✓
```

## 四、关键点总结

| 要点 | 说明 |
|------|------|
| **大整数类型** | 使用 `cpp_int` 处理超大整数 |
| **取余** | `n % -2` 得到余数（-1, 0, 或 1） |
| **求商** | `n /= -2`（向零取整） |
| **余数调整** | `rem < 0` 时，`rem += 2`, `n += 1` |
| **数码记录** | 每次循环将余数（0 或 1）追加到结果 |
| **反转** | 循环结束后反转字符串 |

## 五、与普通整数版本对比

| 普通整数版本 | cpp_int 版本 |
|-------------|--------------|
| `long long n` | `cpp_int n(s)` |
| `int rem = n % -2` | `cpp_int rem_cpp = n % -2` |
| `n /= -2` | `n /= -2` |
| 直接使用 | 需要 `convert_to<int>()` |

**核心逻辑完全相同，只是数据类型换成了大整数！**
