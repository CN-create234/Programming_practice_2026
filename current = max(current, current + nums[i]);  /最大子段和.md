## 最大子段和：
```cpp
给定一个整数序列（可能包含负数），求其连续子段（子数组）的最大和。

示例：[-2, 1, -3, 4, -1, 2, 1, -5, 4] → 最大子段和为 6（对应子段 [4, -1, 2, 1]）

int maxSubArray(vector<int>& nums) {          // 核心算法：求最大子段和，参数为整数数组的引用
    if (nums.empty()) return 0;               // 边界情况：如果数组为空，直接返回0
    
    int current = nums[0];                    // current：以当前位置结尾的最大子段和，初始化为第一个元素
    int best = nums[0];                       // best：全局最大子段和，初始化为第一个元素
    
    for (int i = 1; i < nums.size(); i++) {   // 从第二个元素开始遍历（索引1到n-1）
        // 状态转移：要么从当前位置重新开始（nums[i]），要么接在之前的子段后面（current + nums[i]）
        current = max(nums[i], current + nums[i]);   // 取两者中较大的作为新的current
        //current = max(current, current + nums[i]);   不连续
        best = max(best, current);                    // 更新全局最优解：取best和current中的较大值
    }
    
    return best;                               // 返回全局最大子段和
}

int main() {                                   // 主函数：程序入口
    int n;                                     // 声明变量n，用于存储数组长度
    cin >> n;                                  // 从标准输入读取数组长度
    
    vector<int> nums(n);                       // 创建大小为n的整型向量（动态数组）
    for (int i = 0; i < n; i++) {              // 循环n次，读取每个元素
        cin >> nums[i];                        // 从标准输入读取第i个元素并存入数组
    }
    
    int result = maxSubArray(nums);            // 调用maxSubArray函数计算结果
    cout << result << endl;                    // 输出结果并换行
    
    return 0;                                  // 返回0表示程序正常结束
}

## 二、核心算法函数 `maxSubArray` 详解

### 1. 函数签名
```cpp
int maxSubArray(vector<int>& nums)
```
- **参数**：`vector<int>& nums` —— 整数数组的引用（避免拷贝，提高效率）
- **返回值**：`int` —— 最大子段和

---

### 2. 空数组处理
```cpp
if (nums.empty()) return 0;
```
- 如果数组为空，直接返回 0
- 实际OJ题目中通常保证数组非空，但这是良好的编程习惯

---

### 3. 核心变量初始化
```cpp
int current = nums[0];  // 以当前位置结尾的最大子段和
int best = nums[0];     // 全局最大子段和
```

| 变量 | 含义 | 初始值 |
| :--- | :--- | :--- |
| `current` | **以当前元素结尾**的子数组的最大和 | 第一个元素本身 |
| `best` | **所有位置**中最大的子数组和 | 第一个元素本身 |

**为什么初始化为 `nums[0]`？**
- 至少有一个元素，所以初始状态就是以第一个元素结尾且包含第一个元素的子段

---

### 4. 核心循环（状态转移）
```cpp
for (int i = 1; i < nums.size(); i++) {
    current = max(nums[i], current + nums[i]);
    best = max(best, current);
}
```

这是整个算法的**灵魂**，我们来拆解每一步：

---

#### 第 i 步的状态转移

**核心思想**：对于每个位置 `i`，我们计算 **以 `nums[i]` 结尾的最大子段和**。

**两种选择**：
1. **从当前位置重新开始**：只取 `nums[i]` 本身
   - 适用场景：`current + nums[i] < nums[i]`，即前面的子段是负贡献
2. **接在之前的子段后面**：`current + nums[i]`
   - 适用场景：`current + nums[i] >= nums[i]`，即前面的子段有正贡献

**状态转移方程**：
```
current = max(nums[i], current + nums[i])
```

---

#### 手动推演示例

以 `nums = [-2, 1, -3, 4, -1, 2, 1, -5, 4]` 为例：

| 步骤 | i | nums[i] | 旧 current | current + nums[i] | 新 current | best | 说明 |
| :--- | :--- | :--- | :--- | :--- | :--- | :--- | :--- |
| 初始 | - | - | -2 | - | -2 | -2 | 初始化 |
| 1 | 1 | 1 | -2 | -1 | **1** | 1 | 重新开始（-2是负贡献）|
| 2 | 2 | -3 | 1 | -2 | **-3** | 1 | 重新开始（-3更小，但必须选）|
| 3 | 3 | 4 | -3 | 1 | **4** | 4 | 重新开始 |
| 4 | 4 | -1 | 4 | 3 | **3** | 4 | 接在后面 |
| 5 | 5 | 2 | 3 | 5 | **5** | 5 | 接在后面 |
| 6 | 6 | 1 | 5 | 6 | **6** | 6 | 接在后面 |
| 7 | 7 | -5 | 6 | 1 | **1** | 6 | 接在后面（虽然变小但仍是正）|
| 8 | 8 | 4 | 1 | 5 | **5** | 6 | 接在后面（无法超过之前的6）|

**最终结果**：`best = 6`，对应子段 `[4, -1, 2, 1]`

---

### 5. 关键理解：`current` 和 `best` 的区别

**`current`**（当前状态）：
- 是一个**动态变化**的值
- 表示**必须包含当前位置**的最大子段和
- 每一轮都会重新计算

**`best`**（全局最优）：
- 是一个**持续更新**的值
- 表示**所有位置中**的最大子段和
- 取所有 `current` 的最大值

**类比理解**：
- `current` 像在**爬山**，记录当前所在位置的高度
- `best` 像在**记录最高峰**，取所有经过位置中的最高点

## 四、算法正确性证明

### 1. 最优子结构
- 以 `i` 结尾的最大子段和，只依赖于以 `i-1` 结尾的最大子段和
- 这是典型的动态规划问题，满足最优子结构

### 2. 无后效性
- 计算 `current[i]` 只需要 `current[i-1]` 和 `nums[i]`
- 之前的决策不影响未来的状态

### 3. 贪心本质
```cpp
current = max(nums[i], current + nums[i]);
```
- 如果 `current + nums[i] < nums[i]`，说明前面的子段全是"累赘"，果断舍弃
- 这是贪心的体现：保留正贡献，舍弃负贡献

## 五、边界情况分析

### 1. 所有元素都是负数
```cpp
nums = [-5, -3, -8, -2]
```
- `current = -5, best = -5`
- i=1: `current = max(-3, -5+(-3)=-8) = -3`, `best = -3`
- i=2: `current = max(-8, -3+(-8)=-11) = -8`, `best = -3`
- i=3: `current = max(-2, -8+(-2)=-10) = -2`, `best = -2`
- **返回 -2**（最接近0的负数，即最大负数）

### 2. 所有元素都是正数
```cpp
nums = [1, 2, 3, 4, 5]
```
- `current = 1, best = 1`
- i=1: `current = max(2, 1+2=3) = 3`, `best = 3`
- i=2: `current = max(3, 3+3=6) = 6`, `best = 6`
- i=3: `current = max(4, 6+4=10) = 10`, `best = 10`
- i=4: `current = max(5, 10+5=15) = 15`, `best = 15`
- **返回 15**（所有元素之和）

### 3. 包含零
```cpp
nums = [-2, 0, 1, -3, 4]
```
- 零对结果无影响，但可以作为"缓冲"
- 不影响算法逻辑

## 六、空间优化：为什么可以用 O(1)？

传统DP需要数组存储所有状态：
```cpp
vector<int> dp(n);
dp[0] = nums[0];
for (int i = 1; i < n; i++) {
    dp[i] = max(nums[i], dp[i-1] + nums[i]);
}
```

**优化原理**：
- `dp[i]` 只依赖于 `dp[i-1]`
- 不需要保存所有历史状态
- 用单个变量 `current` 代替整个数组

**这是典型的滚动数组优化**。

## 七、常见误区

### ❌ 错误1：忘记处理空数组
```cpp
// 错误：如果 nums 为空，nums[0] 会越界
int current = nums[0];
```
✅ **正确**：先判断 `if (nums.empty()) return 0;`

### ❌ 错误2：初始化 best = 0
```cpp
int best = 0;  // 错误！
```
- 如果所有元素都是负数，best 会错误地返回 0
- ✅ **正确**：`int best = nums[0];`

### ❌ 错误3：理解偏差
```cpp
// 这样写可以，但不易理解
current = current + nums[i];
if (current < nums[i]) current = nums[i];
```
- 等价于 `current = max(nums[i], current + nums[i])`
- 但写在一起更清晰
