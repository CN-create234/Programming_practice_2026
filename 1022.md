好的，下面我给你**逐行详细解释**输入 `1` 时，代码是如何一步步执行的，特别解释 `char& c` 为什么能直接修改字符串。

---

## 一、完整代码（用于解释）

```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
    int n;
    cin >> n;

    if (n == 0) {
        cout << "0" << endl;
        return 0;
    }

    string s = "0";

    for (int i = 1; i <= n; i++) {
        string t = s;
        for (char& c : t) {
            c = (c == '0') ? '1' : '0';
        }
        s = s + t;
    }

    cout << s << endl;
    return 0;
}
```

---

## 二、逐行执行过程（输入 `1`）

### 第 1 行
```cpp
#include <bits/stdc++.h>
```
- 包含所有标准库（iostream、string、algorithm 等）

---

### 第 2 行
```cpp
using namespace std;
```
- 使用标准命名空间，可以直接写 `cin`、`cout`、`string`

---

### 第 3～4 行
```cpp
int main() {
    int n;
    cin >> n;
```
- 声明整数变量 `n`
- 从键盘读取输入 → 假设输入 `1`，所以 `n = 1`

---

### 第 6～9 行（判断 n == 0）
```cpp
if (n == 0) {
    cout << "0" << endl;
    return 0;
}
```
- `n = 1`，不满足 `n == 0`，跳过

---

### 第 11 行
```cpp
string s = "0";
```
- 定义字符串 `s`，初始化为 `"0"`
- 此时内存中：`s = "0"`

---

### 第 13 行（外层循环）
```cpp
for (int i = 1; i <= n; i++) {
```
- `n = 1`，条件 `1 <= 1` 成立，进入循环
- 这是**唯一一次循环**

---

### 第 14 行
```cpp
string t = s;
```
- 复制 `s` 到 `t`
- 此时：`t = "0"`（和 `s` 内容相同，但各自独立）

---

### 第 15 行（内层循环）
```cpp
for (char& c : t) {
```
- 遍历 `t` 中的每一个字符
- `char& c` 表示 `c` 是 `t` 中字符的**引用**
- 当前 `t = "0"`，只有 1 个字符 `'0'`
- `c` 指向 `t[0]`（内存中实际存储 `'0'` 的位置）

---

### 第 16 行（修改字符）
```cpp
c = (c == '0') ? '1' : '0';
```
- 当前 `c` 是 `'0'`
- 条件 `c == '0'` 成立 → 结果为 `'1'`
- **因为 `c` 是引用，修改 `c` 会直接修改 `t[0]`**
- 此时内存中 `t` 变成 `"1"`

---

### 内层循环结束
- `t` 中所有字符遍历完毕
- `t` 已经从 `"0"` 变成 `"1"`

---

### 第 17 行
```cpp
s = s + t;
```
- 当前 `s = "0"`，`t = "1"`
- `s + t` → `"0" + "1"` → `"01"`
- 赋值给 `s`，此时 `s = "01"`

---

### 外层循环结束
```cpp
i++ → i = 2
检查条件 `2 <= 1` 不成立，退出循环
```

---

### 第 19～20 行
```cpp
cout << s << endl;
return 0;
```
- 输出 `s` → 屏幕上显示 `01`
- 程序结束

---

## 三、图解 `char&` 如何修改 `t`

```
初始状态：
t = "0"

内存布局：
t → [ '0' ]   (地址 0x100)

执行 for (char& c : t)：
c 是引用，指向 t[0] 的地址 0x100

执行 c = '1'：
通过地址 0x100 直接修改内存
t → [ '1' ]

修改完成！
```

---

## 四、如果不用 `char&` 会怎样

```cpp
for (char c : t) {   // 没有 &
    c = '1';         // 改的是副本
}
```
- `c` 是 `'0'` 的**副本**
- 修改 `c` 不会影响 `t`
- `t` 仍然保持 `"0"`

---

## 五、总结表

| 写法 | 含义 | 能否修改原字符串 |
|------|------|------------------|
| `char& c : t` | c 是引用，指向原字符 | ✅ **能** |
| `char c : t`  | c 是副本，独立拷贝 | ❌ 不能 |

---

## 六、一句话总结

**`char& c : t` 中，`c` 是 `t` 中字符的“别名”，直接指向内存中的真实字符，所以修改 `c` 就等于修改 `t` 本身。**
