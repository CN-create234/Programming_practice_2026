# 1021代码 - 逐行详细解释

---

## 一、完整代码

```cpp
#include <bits/stdc++.h>
using namespace std;

string toLower(const string& s) {
    string res = s;
    for (char& c : res) {
        c = tolower(c);
    }
    return res;
}

bool isIgnored(const string& word) {
    string lower = toLower(word);
    return lower == "the" || lower == "a" || lower == "an" ||
           lower == "of" || lower == "for" || lower == "and";
}

int main() {
    int T;
    cin >> T;
    cin.ignore();

    for (int caseNum = 0; caseNum < T; caseNum++) {
        string line;
        getline(cin, line);

        int count = 0;
        string word = "";

        for (char ch : line) {
            if (ch == ' ') {
                if (!word.empty() && !isIgnored(word)) {
                    count++;
                }
                word = "";
            } else {
                word += ch;
            }
        }

        if (!word.empty() && !isIgnored(word)) {
            count++;
        }

        cout << "case #" << caseNum << ":" << endl;
        cout << count << endl;
    }

    return 0;
}
```

## 二、逐行详细解释

### 第一部分：头文件和命名空间

```cpp
#include <bits/stdc++.h>
using namespace std;
```

| 代码 | 含义 |
|------|------|
| `#include <bits/stdc++.h>` | 万能头文件，包含所有 C++ 标准库（iostream, string, cctype 等） |
| `using namespace std` | 使用标准命名空间，可以直接写 `cout`、`cin`、`string`、`tolower` 等，不用加 `std::` 前缀 |

---

### 第二部分：toLower 函数

```cpp
string toLower(const string& s) {
    string res = s;
    for (char& c : res) {
        c = tolower(c);
    }
    return res;
}
```

**逐行解释：**

| 行 | 代码 | 含义 |
|----|------|------|
| 1 | `string toLower(const string& s)` | 函数名 `toLower`，参数是 `const string&`（常量引用，不拷贝，提高效率），返回值是 `string` |
| 2 | `string res = s` | 创建 `res` 字符串，初始化为 `s` 的副本 |
| 3 | `for (char& c : res)` | 范围 for 循环，遍历 `res` 中的每个字符，`char&` 是**引用**，可以修改原字符 |
| 4 | `c = tolower(c)` | 将当前字符转为小写，`tolower` 是 C 标准库函数（在 `<cctype>` 中） |
| 5 | `return res` | 返回转换后的字符串 |

**示例：**
```
toLower("Hello") → "hello"
toLower("THE") → "the"
toLower("An") → "an"
```

**为什么用 `const string&` 而不是 `string s`？**
- `const string&`：不拷贝原字符串，只传递引用，高效
- `const`：保证不会修改原字符串

**为什么用 `char&` 而不是 `char`？**
- `char&`：引用，可以修改 `res` 中的字符
- `char`：值拷贝，修改不会影响原字符

---

### 第三部分：isIgnored 函数

```cpp
bool isIgnored(const string& word) {
    string lower = toLower(word);
    return lower == "the" || lower == "a" || lower == "an" ||
           lower == "of" || lower == "for" || lower == "and";
}
```

**逐行解释：**

| 行 | 代码 | 含义 |
|----|------|------|
| 1 | `bool isIgnored(const string& word)` | 函数名 `isIgnored`，参数是单词的常量引用，返回 `bool` |
| 2 | `string lower = toLower(word)` | 将单词转为小写，存入 `lower`（大小写不敏感） |
| 3 | `return lower == "the" || lower == "a" || ...` | 判断 `lower` 是否在忽略词列表中，是则返回 `true`，否则 `false` |

**忽略词列表：** `the`, `a`, `an`, `of`, `for`, `and`

**示例：**
```
isIgnored("the") → lower="the" → true（忽略）
isIgnored("The") → lower="the" → true（忽略）
isIgnored("is") → lower="is" → false（不忽略）
isIgnored("a") → lower="a" → true（忽略）
isIgnored("sample") → lower="sample" → false（不忽略）
```

### 第四部分：主函数开始

```cpp
int main() {
    int T;
    cin >> T;
    cin.ignore();
```

**逐行解释：**

| 行 | 代码 | 含义 |
|----|------|------|
| 1 | `int main()` | 主函数入口 |
| 2 | `int T` | 声明整数变量 `T`，存储测试用例数量 |
| 3 | `cin >> T` | 读取第一行的整数，存入 `T` |
| 4 | `cin.ignore()` | 忽略第一行末尾的换行符，防止后续 `getline` 读到空行 |

**为什么需要 `cin.ignore()`？**

```
输入：
2          ← cin >> T 读取 2，但换行符 '\n' 还留在缓冲区
This is a sample  ← 如果没有 ignore，getline 会先读到空行！
```

`cin.ignore()` 读取并丢弃一个字符（换行符），确保 `getline` 从正确位置开始。

### 第五部分：循环处理每组测试数据

```cpp
    for (int caseNum = 0; caseNum < T; caseNum++) {
        string line;
        getline(cin, line);
```

**逐行解释：**

| 行 | 代码 | 含义 |
|----|------|------|
| 1 | `for (int caseNum = 0; caseNum < T; caseNum++)` | 循环 `T` 次，`caseNum` 从 0 开始（输出格式要求 `case #0:`） |
| 2 | `string line` | 声明字符串变量，存储当前行的输入 |
| 3 | `getline(cin, line)` | 读取一行（包含空格）存入 `line`，遇到换行符停止 |

**`getline` vs `cin >>`：**

| 方式 | 读取内容 | 空格 |
|------|----------|------|
| `cin >> line` | 读取到空格或换行符停止 | ❌ 会丢失空格 |
| `getline(cin, line)` | 读取到换行符停止 | ✅ 保留空格 |

题目中的字符串包含空格（单词之间），所以必须用 `getline`。

# 可以不用读取整行吗？

**答案：可以，但不建议，而且会出问题！**

---

## 一、为什么题目要求用 `getline` 读取整行？

### 输入格式

```
第 1 行：整数 T
第 2 ~ T+1 行：每一行是一串由英文单词和空格组成的字符串
```

**关键点：字符串中包含空格！**

```
This is a sample
↑      ↑  ↑ ↑
空格   空格 空格 空格
```

如果不用 `getline`，用 `cin >>` 读取，会遇到问题。

---

## 二、`cin >>` 的问题

### `cin >>` 默认以空格作为分隔符

```cpp
string word;
cin >> word;  // 读取到空格就停止！
```

对于输入 `"This is a sample"`：
- 第一次 `cin >> word` → 读取 `"This"`
- 第二次 `cin >> word` → 读取 `"is"`
- 第三次 `cin >> word` → 读取 `"a"`
- 第四次 `cin >> word` → 读取 `"sample"`

**问题：** 你不知道一行有多少个单词，无法判断什么时候该换行！

---

## 三、用 `cin >>` 的替代方案

### 方案1：读取所有单词，不关心行

```cpp
int T;
cin >> T;

for (int caseNum = 0; caseNum < T; caseNum++) {
    int count = 0;
    string word;

    // 问题：如何知道一行的单词读完了？
    // 无法判断！cin >> 不知道换行符的存在
    while (cin >> word) {
        // 会一直读到 EOF，无法按行分割
    }
}
```

**不可行！** 因为 `cin >>` 会忽略换行符，无法区分不同行的数据。

---

### 方案2：先读取整行，再用 `stringstream` 分割

这是我们采用的方法：

```cpp
getline(cin, line);           // 读取整行
stringstream ss(line);        // 放入字符串流
while (ss >> word) {          // 从字符串流中读取单词
    // 处理 word
}
```

---

## 四、能不能不用 `getline`，直接用 `cin >>` 然后计数？

### 可以！但需要额外的行信息

如果输入格式改为：

```
2
4
This is a sample
7
Love is a lamp while friendship is the shadow
```

即每行先给出单词数量，那么可以用 `cin >>`：

```cpp
int T;
cin >> T;

for (int caseNum = 0; caseNum < T; caseNum++) {
    int n;
    cin >> n;  // 读取单词数量

    int count = 0;
    for (int i = 0; i < n; i++) {
        string word;
        cin >> word;
        if (!isIgnored(word)) count++;
    }

    cout << count << endl;
}
```

**但题目没有给出单词数量，所以这种方法不可行！**

---

## 五、为什么必须用 `getline`？

| 原因 | 说明 |
|------|------|
| **保留空格** | `getline` 保留空格，`cin >>` 会丢失空格 |
| **按行分割** | 每组数据是一行，`getline` 正好按行读取 |
| **知道行结束** | `getline` 读取到换行符停止，正好是一行结束 |
| **无法用 `cin >>` 判断行尾** | `cin >>` 忽略换行符，无法知道一行何时结束 |

---

## 六、两种读取方式对比

| 场景 | 使用 `cin >>` | 使用 `getline` |
|------|--------------|----------------|
| 读取整数 | ✅ 可以 | ❌ 会读到换行符 |
| 读取不含空格的字符串 | ✅ 可以 | ✅ 可以 |
| 读取包含空格的字符串 | ❌ 只读第一部分 | ✅ 读取整行 |
| 按行分割数据 | ❌ 无法判断行尾 | ✅ 天然按行分割 |

---

## 七、总结

| 问题 | 答案 |
|------|------|
| 可以不用 `getline` 吗？ | 可以，但需要知道每行的单词数量 |
| 题目给了单词数量吗？ | ❌ 没有 |
| 如何判断一行结束？ | 只有 `getline` 能做到 |
| 正确做法是什么？ | `getline` 读取整行 + `stringstream` 分割 |

**核心：因为输入以行为单位，且行内包含空格，必须用 `getline` 读取整行，再按空格分割单词。**
